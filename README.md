# KOSDAQ 중소형주 자동매매 시스템

한국투자증권 Open API(KIS Developers)를 활용한 **KOSDAQ 중소형주 단기 자동매매 봇**입니다.
VWAP · 볼린저밴드 기반 스크리닝과 실시간 체결 데이터 분석을 결합한 역추세 + 모멘텀 복합 전략을 사용합니다.

> 모의투자 환경에서 개발 및 테스트되었습니다.

---

## 전략 개요

### 핵심 아이디어

> "전일 기준 가격 밴드 아래로 눌린 종목을 선별해 두고, 장중 매수세 폭발 시점에 진입한다."

- **역추세 필터**: 전일 VWAP·BB 중간선보다 낮은 종목만 후보로 선정 → 과매도 구간 진입 노린다
- **모멘텀 트리거**: 거래량 급증 + 체결강도 우위 + 매도잔량 소진이 동시에 발생할 때만 매수 → 단순 눌림이 아닌 실제 매수세 전환 확인 후 진입
- **빠른 손절**: -2% 손절 / 고점 대비 -1.5% 트레일링 스탑으로 손실 제한
- **BB 상단 홀드**: BB 상단 터치 구간에서는 트레일링 스탑을 비활성화해 추세 연장 구간 수익 극대화

### 전략 플로우

```
장 시작 전
  └─ 코스닥 시가총액 500억~5000억 종목 전체 조회
        └─ 스크리닝 (VWAP 조건 + BB 조건)
              └─ 종합점수 계산 → 점수 비례 자금 배분
                    └─ 시가 / 전일종가 조회

장 중 (WebSocket 실시간)
  └─ 체결 데이터 수신 (H0STCNT0)
  └─ 호가창 데이터 수신 (H0STASP0)
        └─ 매수 시그널 감지 → 매수 주문
              └─ 보유 중: 손절 / 트레일링 스탑 감시

15:20 / 15:30
  └─ 미청산 포지션 전량 강제 청산
```

## 매수 진입 조건 (전체 AND)

### Phase 1 — 스크리닝 (`screener.py`)

| 조건 | 기준 | 의도 |
|------|------|------|
| **종목 풀** | 코스닥 시가총액 500억~5000억 | 유동성 있는 중소형주, 대형주·초소형주 제외 |
| **VWAP 조건** | 현재가 < 전일 VWAP `(H+L+C)/3` | 전일 평균 매매단가 아래 → 단기 저평가 구간 |
| **BB 조건** | 현재가 < 볼린저밴드 20일 중간선 | 중기 추세 평균 아래 → 추가 눌림 여지 확인 |

통과 종목은 **종합점수**로 자금 배분 비중 결정:
- VWAP 괴리율 점수 (50%) + BB 위치 점수 (50%)
- VWAP 대비 더 낮을수록, BB 하단에 더 가까울수록 높은 점수 → 눌림이 깊을수록 더 많은 자금 배분

### Phase 2 — 실시간 매수 시그널 (`trader.py`)

| 조건 | 기준 | 의도 |
|------|------|------|
| **갭 필터** | 시가 갭 ≤ +10% | 갭업 과열 종목 진입 방지 |
| **데이터 워밍업** | 최소 30틱 수신 후 판단 | 초반 노이즈 제거 |
| **거래량 급증** | 현재 체결량 ≥ 직전 평균 × **2배** | 단순 눌림이 아닌 관심 급증 시점 포착 |
| **양봉 방향** | 현재가 > 시가 (등락률 > 0%) | 하락 중 반등 확인 |
| **체결강도** | 매수체결량 합계 / 매도체결량 합계 ≥ **110%** | 매수세가 매도세 압도하는지 확인 |
| **매도잔량 소진** | 최우선 매도호가 잔량 < 직전 평균의 **30%** | 매도벽 붕괴 → 가격 저항 제거 확인 |

6개 조건 **전부 충족** 시 매수 주문 실행.

## 매도 / 청산 조건

| 조건 | 기준 | 비고 |
|------|------|------|
| **손절** | 매수가 대비 **-2%** | 즉시 매도 |
| **트레일링 스탑** | 고점 대비 **-1.5%** | BB 상단(±0) 터치 구간에서는 홀드 |
| **강제 청산** | **15:20** 도달 시 | WebSocket 종료 후 전량 청산 |
| **장마감 청산** | **15:30** 이후 | 미청산 포지션 잔고 조회 후 전량 청산 |

## 포지션 사이징

- 스크리닝 통과 종목들의 종합점수 합 대비 개별 점수 비율로 자금 배분
- 예: 종목 A 점수 0.6 / 종목 B 점수 0.4 → A에 60%, B에 40%
- 눌림이 깊고 BB 하단에 가까울수록 더 많은 자금 할당

## 백테스트 (`backtest.py`)

- 기간: 최근 90일 KOSDAQ 중소형주 1분봉
- 데이터: 한투 Open API 분봉 + pykrx / FinanceDataReader 시총 필터
- 분봉 BB(20, 2σ) + 거래량 급증(2배) 조건 재현
- 손절 -3%, 최대 보유 120분 기준
- 결과는 `backtest_result.csv`로 저장

## 설치 및 실행

### 1. 환경변수 설정
```bash
cp .env.example .env
# .env 파일에 실제 키 입력
```

```env
APP_KEY=your_app_key
APP_SECRET=your_app_secret
ACCOUNT_NO=your_account_number
ACCOUNT_NO_SEQ=01
```

### 2. 의존성 설치
```bash
python -m venv venv
source venv/bin/activate
pip install requests numpy pandas websocket-client python-dotenv pykrx FinanceDataReader
```

### 3. 실행
```bash
python main.py
```

### 4. cron 자동 실행 예시
```cron
30 8 * * 1-5 /home/user/trading/run.sh   # 평일 08:30 자동 시작
```
